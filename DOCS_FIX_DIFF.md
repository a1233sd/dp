# Правки документов под фактическую программу

Цель файла: дать список ручных правок для `.docx`, чтобы ТЗ, ПМИ, календарный план и задание не противоречили программе. Формат: что сейчас написано, что нужно заменить, и грубый diff.

Фактическая программа: локальный FastAPI-сервис со Swagger UI (`/docs`). Основная работа выполняется через HTTP API: создание пользователей, добавление документов, загрузка PDF, запуск проверки, получение результата и отчета.

### 1.5. Исправить приложение со списком HTTP-методов

Раздел: `ПРИЛОЖЕНИЕ Г СПИСОК HTTP МЕТОДОВ`.

Нужно привести таблицу к фактическому API программы.

Должно быть:

| Метод | URL | Назначение |
|---|---|---|
| `POST` | `/users` | Создание пользователя |
| `GET` | `/users` | Получение списка пользователей |
| `POST` | `/documents` | Создание документа из текста |
| `POST` | `/documents/upload` | Загрузка PDF-документа |
| `GET` | `/documents` | Получение списка документов |
| `GET` | `/documents/{document_id}` | Получение документа по ID |
| `PATCH` | `/documents/{document_id}` | Обновление документа |
| `DELETE` | `/documents/{document_id}` | Удаление документа |
| `GET` | `/archive/unique` | Получение архива уникальных работ |
| `POST` | `/documents/{document_id}/archive` | Добавление документа в архив уникальных работ |
| `POST` | `/rules/exclusions` | Создание правила исключения |
| `GET` | `/rules/exclusions` | Получение списка правил исключения |
| `DELETE` | `/rules/exclusions/{rule_id}` | Удаление правила исключения |
| `POST` | `/checks` | Запуск проверки на заимствования |
| `GET` | `/checks/{check_id}` | Получение сохранённого результата проверки |
| `PATCH` | `/checks/{check_id}/originality` | Изменение процента оригинальности |
| `GET` | `/checks/{check_id}/report` | Получение структурированного отчёта |

Отдельно можно не включать в таблицу Swagger следующие служебные методы, потому что они скрыты из OpenAPI-схемы:

| Метод | URL | Назначение |
|---|---|---|
| `GET` | `/health` | Проверка состояния сервиса |
| `GET` | `/settings` | Получение настроек по умолчанию |

Критичные замены:

```diff
-- POST /archive/unique - Получение архива уникальных работ
+ GET /archive/unique - Получение архива уникальных работ

-- GET /rules/exclusions - Удаление правила по ID
+ DELETE /rules/exclusions/{rule_id} - Удаление правила исключения

+ PATCH /documents/{document_id} - Обновление документа
+ DELETE /documents/{document_id} - Удаление документа
+ PATCH /checks/{check_id}/originality - Изменение процента оригинальности
+ POST /documents/{document_id}/archive - Добавление документа в архив уникальных работ
```

Также лучше убрать обёртки вида `{ document: DocumentOut }`, `{ check: CheckOut }`, `{ rule: ExclusionRuleOut }`, если они есть в таблице. Фактически FastAPI возвращает сами модели ответа напрямую.

## 5. Краткий чек-лист после правок

- В ТЗ нет требования разграничения доступа.
- В ТЗ и ПМИ указаны PDF-документы с текстовым слоем, а не абстрактные распространённые форматы.
- В ПМИ все действия выполняются через Swagger UI по адресу `http://127.0.0.1:8000/docs`.
- В ТЗ приложение HTTP-методов совпадает с фактическими endpoint'ами программы.
- В КП указан правильный студент.
- Во всех документах одинаковые группа, руководитель и тема.
- Опечатка `заимстований` исправлена на `заимствований`.
